# The End of Conflict Framework

A five-stage process for analysing and resolving conflicts—external disputes or internal dilemmas.

**Core principle:**  
Name it → See it → Judge it → Choose it → Do it.

## The Journey
```
Map → Truth → Judgment → Decision → Action
```

| Stage     | Activity                  | Output                     | Why it matters (what happens if skipped) |
|-----------|---------------------------|----------------------------|------------------------------------------|
| **Map**   | Name and structure        | Clear structure of parties and conflict | Without boundaries, later stages drift or solve the wrong problem |
| **Truth** | Establish reality         | Verified facts, events, and real needs/interests | Arguments persist on different narratives |
| **Judgment** | Evaluate what went wrong | Assessment of causes and accountability | You can't fix what you don't understand caused it |
| **Decision** | Choose resolution         | Specific intention/commitment | Knowing what's wrong ≠ deciding to act |
| **Action** | Implement                 | Actual change              | Most conflicts die here—talk without execution |

**Core questions to move forward:**
- **Map:** Who are the parties? What is the conflict?
- **Truth:** What happened? What is real? What are the needs and interests?
- **Judgment:** What went wrong? What must be addressed? Who is accountable?
- **Decision:** How do we resolve the conflict we mapped?
- **Action:** How do we put the resolution into practice?

## When to Use This Framework
Use when:
- There are ≥2 genuine opposing positions
- Each position is held by someone who can choose/act
- You can name the conflict in one sentence
- You know enough to sketch an initial map

Do **not** use when:
- Only frustration (no second position)
- The "other side" is nature or the irreversible past
- You can't yet define what the fight is about

## Before You Begin: Locate Yourself
Two dimensions set your coordinates.

### 1. Position — Your relationship to the conflict
| Position       | Description                                      | Implications                              |
|----------------|--------------------------------------------------|-------------------------------------------|
| **Observer**   | No power to affect outcome                       | Pure analysis; easier objectivity         |
| **Party**      | You can affect outcome (in it or have authority) | Blind spots likely; harder to see clearly |
| **Internal**   | Conflict within yourself                         | You are analyst + all parties             |

### 2. Time — When is the conflict?
| Time     | Evidence available       | Action possible? |
|----------|--------------------------|------------------|
| **Past** | Memory, documents        | Learn only       |
| **Present** | Real events, claims      | Full framework   |
| **Future** | Projections, scenarios   | Simulation only  |

## Stage 1: MAP – Define the Conflict

**Objective:** Draw the map before analysing it.  
**Why:** Undefined conflicts produce drifting analysis.

### Structure: Name → Categorise → Type

#### A. Parties
| Move      | Action                                      | Questions                                   |
|-----------|---------------------------------------------|---------------------------------------------|
| Name      | List everyone connected                     | Who fights visibly? Who influences quietly? |
| Categorise| Main (visible actors) / Sub (hidden influencers) |                                             |
| Type      | Benefits / Loses / Both / ? from continuation | What do they gain/lose if it continues vs ends? |

**Template:**
| Party          | Level (Main/Sub) | Type (Benefits/Loses/Both/?) | Notes                  |
|----------------|------------------|------------------------------|------------------------|

#### B. Conflict
| Move      | Action                                      | Guidance                                    |
|-----------|---------------------------------------------|---------------------------------------------|
| Name      | One-sentence main conflict                  | Newspaper headline test                     |
| Categorise| List 3–6 specific sub-conflicts             | Concrete, pointable disagreements           |
| Type      | Operational (what to do) / Structural (who decides) / Both | Recurring fights = likely Structural        |

**Template:**
| Main Conflict | [One sentence] |
|---------------|----------------|
| Sub-conflict  | Type           |
| ...           | ...            |

**Output checklist:** All parties and conflicts named, categorised, typed.

**Link forward:** This map tells you *who* and *what*—but not what is true. Stage 2 establishes reality.

## Stage 2: TRUTH – Establish Reality

**Objective:** Separate fact from narrative in each sub-conflict.  
**Input:** Sub-conflicts from Map.  
**Process:** Complete CAPTURE → TEST → INFER → CHECK for each sub-conflict.

### 1. CAPTURE (SAY vs DO)
| Party | SAY (claims)       | Evidence | DO (actions)       | Evidence |
|-------|--------------------|----------|--------------------|----------|
|       |                    |          |                    |          |

### 2. TEST
- **Accuracy:** Actions true? Facts true?
- **Alignment:** Do actions serve stated goal?
- If inaccurate: Error / Bias / Partial info / Deceit?

### 3. INFER
Decision tree: Stated goal? → Alignment? → Candidate real want from actions.

| Party | Candidate Want | Confidence (High/Med/Low) | Mismatch reason |
|-------|----------------|---------------------------|-----------------|

### 4. CHECK (bias guard)
Answer:
- Evidence-based?
- Missing evidence?
- Disconfirming evidence?
- Generous alternative?
- Equal standards?
- **My own want?**

**Output:** Verified reality + candidate real wants + your bias acknowledged.

**Link forward:** You now know what is real—but not what is right. Stage 3 evaluates.

## Stage 3: JUDGMENT – Evaluate What Went Wrong

**Objective:** Identify causes and accountability.  
**Input:** Verified truth from Stage 2.

For each sub-conflict:
1. What **should** have happened? (rules, agreements, expectations)
2. What **did** happen?
3. Gap?
4. Cause? (Deception / Concealment / Neglect / Mistakes / Miscommunication)
5. Who responsible?
6. What must be addressed?

**Template (per sub-conflict):**
| Sub-conflict | Should | Did | Gap | Cause | Responsible | Must Address |
|--------------|--------|-----|-----|-------|-------------|--------------|

**Categories evaluated:**
- Deception (lied about)
- Concealment (hidden)
- Neglect (needs ignored)
- Mistakes
- Miscommunication
- Accountability

**Output:** Clear assessment of violations, failures, unmet needs, accountability.

**Link forward:** You know what went wrong—but not how to fix it. Stage 4 chooses.

## Stage 4: DECISION – Choose Resolution

**Objective:** Select a right, wise, doable path.  
**Input:** Judgment (what must be addressed) + candidate wants.

Test proposed resolutions using hierarchy:

1. **L.E.R** (RIGHT?) → Legal / Ethical / Reversible (Mirror check)
2. **B.U.S** (WISE?) → Benefit / Universal / Sustainable
3. **A.C.T** (DOABLE?) → Assets / Capability / Timely

**Critical rule:** Reject anything failing L.E.R or B.U.S, even if easy (A.C.T passes).

**Template:**
| Proposed Resolution | L.E.R | B.U.S | A.C.T | Verdict |
|---------------------|-------|-------|-------|---------|

**Output:** Chosen commitment—what will actually be done.

**Link forward:** Intention without execution changes nothing. Stage 5 acts.

## Stage 5: ACTION – Implement

**Objective:** Create real change.  
**Tools:** Concrete steps, owners, timelines, follow-up.

Most conflicts die here. Action closes the loop—or reveals the map was incomplete (restart).
